## Content Insights
I learned that hosting on GitHub Pages includes case sensitivity that local Windows environments don't typically enforce. If my file is named Sydney.jpg but my CSS calls for sydney.jpg, the site will break. I also discovered how GitHub handles subfolder routing. Because my index.html is in a week_1 folder, the URL must explicitly include the folder path.<br> 
I also learned the efficiency of using embedded Unsplash URLs rather than local files. Using embed images made it significantly easier to swap around images to suite my web design without having to download them to my device. However, I also learned that this requires using object-fit: cover in my CSS. Without it, the image would stretch or squish to fit the 1:1 square ratio of my venue cards, ruining the professional look of the Explore page.

## Career/Employability/Learning Insights
My workflow also became significantly more time and space efficient once I swapped manual file management for the Unsplash API. Learning to embed images allowed me to iterate on the "Coeliac Compass" design much faster. This taught me that professional development isn't just about coding from scratch, it's about knowing which technical shortcuts to use without sacrificing quality.<br> 
Finally, I found that I learn best through immediate problem-solving. Troubleshooting the case-sensitivity issue on GitHub Pages taught me more about server behaviour than a standard tutorial ever could. I’ll continue using this strategy to stay engaged and ensure I actually understand the why behind the code.
